Position

Elements Explaination Example
position position: static|absolute|fixed|relative|sticky|initial|inherit; specify the type of positioning method used for an element

position: static;

    relative to the normal flow of the page.
  • top, right, bottom, left: no effect
  • not act as anchor point for the absolute child
  • not removed from normal flow

position: relative;

    relative to its original position.
  • top, right, bottom, left: have effect
  • act as anchor point for the absolute child
  • not removed from normal flow
    => leave the gap unfilled | can overlap elements

position: absolute;

relative to the nearest positioned ancestor.
  • top, right, bottom, left: have effect
  • act as anchor point for the absolute child
  • removed from normal flow
    => leave the gap filled by next elements | can overlap elements

position: fixed;

relative to the viewport (always in the same place when scrolled)
  • top, right, bottom, left: have effect
  • act as anchor point for the absolute child
  • removed from normal flow
    => leave the gap filled by next elements | can overlap elements

position: sticky;

    toggle between relative and fixed
    relative until a given offset position is met
  • top, right, bottom, left: have effect
  • act as anchor point for the absolute child
  • removed from normal flow
    => leave the gap filled by next elements | can overlap elements

more specific

top: auto|length|initial|inherit; right: auto|length|initial|inherit; bottom: auto|length|initial|inherit; left: auto|length|initial|inherit;
z-index z-index: auto|number|initial|inherit; specify the stack order of an element
auto: the last code have higher z-index

clear clear: none|left|right|both|initial|inherit;
specify what should happen with the next element
After using float property, we want the next element continue to flow, but below (not on right or left)
Should match the clear to the float
  • left - The element is pushed below left floated elements
  • right - The element is pushed below right floated elements
  • both - The element is pushed below both left and right floated elements

Clearfix

Problem

the floated element is taller than the container

Solution 1

use overflow property (no need for clear property)
scrollbars may appear if not controlling well paddings and margins

                                }

Solution 2

use the modern clearfix hack (no need for clear property) 
                                .clearfix::after {
                                    content: "";
                                    clear: both;
