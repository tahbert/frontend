CSS Position

Elements Explaination Visual Code
position position: static|absolute|fixed|relative|sticky|initial|inherit;
Specify the type of positioning method used for an element see more
position: static;
Relative to the normal flow of the page.
  • top, right, bottom, left: no effect
  • Not act as anchor point for the absolute child
  • Not removed from normal flow
position: relative;
Relative to its original position.
  • top, right, bottom, left: have effect
  • Act as anchor point for the absolute child
  • Not removed from normal flow => leave the gap unfilled | Can overlap elements
position: absolute;
Relative to the nearest positioned ancestor.
  • top, right, bottom, left: have effect
  • Act as anchor point for the absolute child
  • Removed from normal flow => leave the gap filled by next elements | Can overlap elements
position: fixed;
Relative to the viewport (always in the same place when scrolled)
  • top, right, bottom, left: have effect
  • Act as anchor point for the absolute child
  • Removed from normal flow => leave the gap filled by next elements | Can overlap elements
position: sticky;
Based on the user's scroll position
(toggles between relative and fixed: relative until a given offset position is met in the viewport - then fixed)
position: -webkit-sticky; /* Safari */
  • top, right, bottom, left: have effect
  • Act as anchor point for the absolute child
  • Removed from normal flow => leave the gap filled by next elements | Can overlap elements

  float: right ;
}

clear clear: none|left|right|both|initial|inherit;
Specify what should happen with the next element
After using float property, we want the next element continue to flow, but below (not on right or left)
Should match the clear to the float
  • left - The element is pushed below left floated elements
  • right - The element is pushed below right floated elements
  • both - The element is pushed below both left and right floated elements
clear: left;
.div1 (contain .div1-1)

  float: left;
}

clear: right;
.div1 (contain .div1-1)
.div1-1 {
  float: right ;
}
.div2 {float: none; clear: right;}
clear: both;
.div1 (contain .div1-1)
.div1-1 {
  float: left ;
}
.div1-2 {
  float: right ;
}
.div2 {float: none; clear: both;}
Clearfix Problem: the floated element is taller than the container
Solution 1: Using overflow property (no need to use clear property)
scrollbars may appear if not controlling well paddings and margins
.clearfix {
  overflow: auto;
}

Solution 2: Using the modern clearfix hack (no need to use clear property)
.clearfix::after {
  content: "";
  clear: both;
  display: table;
}
