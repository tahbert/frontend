Events

Keywords Explaination Example
Interface: Event
waiting when the media stops to buffer the next frame.
most used on <video>
cancelable | bubble
el.onwaiting = () => {
 alert('please wait!');
};
abort when the media data download has been aborted, not because of an error.
cancelable | bubble
el.onabort = () => {
 alert('abort loading!');
};
volumechange each time the volume has been changed.
(increase/ decrease/ mute/ unmute)
cancelable | bubble
el.onvolumechange = () => {
 alert('volume changed!');
};
timeupdate when the playing position has changed.
cancelable | bubble
el.ontimeupdate = () => {
 alert('playing position changed!');
};
seeking when the user starts moving/skipping to a new position.
cancelable | bubble
el.onseeking = () => {
 alert('seek operation began!');
};
seeked when the user is finished moving/skipping to a new position
cancelable | bubble
el.onseeked = () => {
 alert('seek operation completed!');
};
error when an error occurred during the loading of a media
cancelable | bubble
el.onerror = () => {
 alert('loadding error!');
};
stalled when the browser is trying to get media data, but data is not available.
cancelable | bubble
el.onstalled = () => {
 alert('data unavailable!');
};
suspend when the browser is intentionally not getting media data.
cancelable | bubble
el.onsuspend = () => {
 alert('data loading suspended!');
};
ended when the media has reached the end
useful for messages like "thanks for watching", etc.
cancelable | bubble
el.onended = () => {
 alert('Thanks for watching!');
};
play when paused property is changed from true to false
when as a result of play() , or autoplay attribute.
cancelable | bubble
el.onplay = () => {
 alert('the video has started to play!');
};
playing after playback is first started, and whenever it is restarted
cancelable | bubble
el.onplaying = () => {
 alert('the video is now playing!');
};
pause when the media is paused either by the user or programmatically.
cancelable | bubble
el.onpause = () => {
 alert('video paused!');
};
ratechange the playing speed of the media is changed (slow motion or fast forward)
cancelable | bubble
el.playbackRate = 0.3;
el.onratechange = () => {
 alert('slow motion!');
};
Interface: Event > During the loading
canplay when the browser can play the media (buffered enough).
cancelable | bubble
el.oncanplay = () => {
 alert('can start playing!');
};
canplaythrough when the browser can play through the media (without stopping for buffering)
cancelable | bubble
el.oncanplaythrough = () => {
 alert('can play through without stopping!');
};
loadstart when the browser starts loading the media.
cancelable | bubble
el.oncanplay = () => {
 alert('starting to load media!');
};
durationchange when the duration of the media is changed
cancelable | bubble
el.ondurationchange = () => {
 alert('duration changed!');
};
loadedmetadata when meta data for the media has been loaded.
cancelable | bubble
el.onloadedmetadata = () => {
 alert('meta data loaded!');
};
loadeddata when data for the current frame is loaded, but not enough data to play next frame
cancelable | bubble
el.onloadeddata = () => {
 alert('current frame loaded!');
};
progress when the browser is downloading the media
cancelable | bubble
el.onprogress = () => {
 alert('downloading video!');
};
