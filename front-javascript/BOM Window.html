Events

Keywords Explaination Example
Interface: Event
load when the whole page has loaded completely
unlike DOMContentLoaded, without waiting for resources to finish loading.
  • most often used within <body> to execute a script
  • can be used to check the browser to load the proper web version
  • can be used to deal with cookies
 window.onload = () => {
 alert('page is fully loaded');
};
beforeunload when the document is about to be unloaded.
most often used to inform whether to stay or leave the page.

the return text can't be changed

window.onbeforeunload = () => {
 return "different in different browsers";
};
unload once a page has unloaded (or the browser window has been closed)
may not always work as expected due to different browser settings
 unnecessary
beforeprint when a page is about to be printed
by Safari, Opera

before the print dialogue box appears

window.onbeforeprint = () => {
 alert("You are about to print the document!");
}
afterprint when a page has started printing
by Safari, Opera

when the print dialogue box has been closed

window.onafterprint = () => {
 alert("This document is now being printed");
}
error when a resource failed to load or couldn't be used
cancelable | bubble
UIEvent if generated from a user interface element, otherwise Event
complex
offline when the browser starts to work offline
by Chrome, Edge, Safari
 unnecessary
online when the browser starts to work online
by Chrome, Edge, Safari
Don't use to determine the availability of a particular website.
unnecessary
Interface: UiEvent
resize when the browser window has been resized.

return window width when resizing

window.onresize = () => {
 alert(window.outerWidth);
}
Interface: MessageEvent
message when the window receives a message
cancelable | bubble
window.onmessage = (event) => {
 console.log(`received message: ${event.data}`);
};
Interface: HashChangeEvent
hashchange when there has been changes to the anchor part (begins with a '#' symbol) of the current URL. el.onclick = (event) => {
 location.hash = "part5";
};
window.onhashchange = () => {
 alert("The anchor part has changed!");
};
Interface: PageTransitionEvent
pageshow when a user navigates to a webpage.
    differ from onload:
  • it occurs after the onload event when the page first loads.
  • it occurs every time the page is loaded
 window.onpageshow = (event) => {
 alert("Welcome to our website!")
}
pagehide when the user is navigating away from a webpage.
sometimes used instead of onunload, as onunload causes the page to not be cached.
may not always work as expected due to different browser settings
 window.onpagehide = (event) => {
 if (event.persisted) {
  /* the page can be reused later */
 }
}
Interface: FocusEvent
blur when the window loses focus
cancelable | bubble
window.onblur = () => {
 console.log("focus out!")
}
focus when the window gets focus
cancelable | bubble
window.onfocus = () => {
 console.log("focus in!")
}
Interface: ClipboardEvent
General original target: Element
=> listen on Window to handle it in the capture or bubbling phases
copy when the user initiates a copy action through the browser's user interface. window.oncopy = (event) => {
 console.log('copy action initiated');
};
cut when the user initiates a cut action through the browser's user interface. window.oncut = (event) => {
 console.log('cut action initiated');
};
paste when the user initiates a paste action through the browser's user interface. window.onpaste = (event) => {
 console.log('paste action initiated');
};
