Const keyword

Elements Explaination Example
When to use always use const keyword unless the value will change
=> use when declare a new array, object, function, regExp
Objects and Arrays const define a constant reference to a value, not a constant value
    => possible
  • change the elements of constant array
  • change the properties of constant object
 const cars = ["Saab", "Volvo", "BMW"]; // constant array
cars[0] = "Toyota"; // change an element:
cars.push("Audi"); // add an element:
const car = {type:"Fiat", model:"500", color:"white"}; // const object
car.color = "red"; // change a property
car.owner = "Johnson"; // add a property
    => impossible
  • reassign a constant value
  • reassign a constant array
  • reassign a constant object
 const car = "Saab";
car = "Volvo"; // error
const cars = ["Saab", "Volvo", "BMW"];
cars = ["Toyota", "Volvo", "Audi"]; // error
const car = {type:"Fiat", model:"500", color:"white"};
car = {type:"Volvo", model:"EX60", color:"red"}; // error

Scope

Elements Explaination Example
Scope determine the accessibility of variables
Variable Lifetime
  • local variables: from declared to function completed
  • global variables: from declared to browser/tab closed
Block Scope variables declared inside a {...} block
not accessible from outside (except with var keyword)
{
 var x = 2;
 let y = 2;
 const z = 2;
}
x;
y;
z;
Local Scope variable declared inside a function
not accessible from outside
 function myFn() {
 var a = 2;
 let b = 2;
 const c = 2;
}
a;
b;
c;
I use different chacracters since x of the prev example can be accessed everywhere on the page
Global scope variables declared outside a function or block
  • accessible from everywhere
  • undeclared variable => automatically global (not in strict mode)
becareful: global variables can cause name conflicts
From Inside every function have access to scope above them => called closure
every block scope have access to scope above them

access outer block scopes
