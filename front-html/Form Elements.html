Attributes for <form>

Elements Explaination Example
action <form action="URL"></form> (default: current page)
Specify where to send the form-data when a form is submitted 		<form action="/action_page.php"></form>
target <form target="_blank|_self|_parent|_top|framename"></form>
Specify where to display the response
_self: same frame (default)
_top: full window
_blank: new tab/window
_parent: parent frame
framename: named iframe
accept-charset <form accept-charset="character_set"></form>
Specify the character encodings for form submission (Most common: UTF-8, ISO-8859-1) 		<form action="/action_page.php" accept-charset="utf-8"></form>
method <form method="get|post"></form>
Specify how to send form-data
  • get: Append the form data to the URL, in name/value pairs (default)
  • post: Append the form data inside the body of the HTTP request
enctype <form enctype="value"></form>
Specify how the form-data should be encoded when submitting it to the server (only for method="post")
  • application/x-www-form-urlencoded: (default)
  • multipart/form-data: necessary if uploading a file
  • text/plain: not encode at all not recommended
autocomplete <form autocomplete="on|off"></form> (default: on)
Specify autocomplete is on or off
possible to have autocomplete "on" for the form, "off" for specific input fields, or vice versa
on
off
name <form name="text"></form>
Specify the form name
Used to reference elements in a JS, or to reference form data after a form is submitted 		<form action="/action_page.php" method="get" name="myForm"></form>
novalidate <form novalidate></form>
Specify that the form should not be validated when submitted		 <form action="/action_page.php" novalidate></form>
rel <form rel="value"></form>
Specify the relationship between the current document and the linked document		 see all here

Other Attributes for <input>

Elements Explaination Example
name required Used to reference elements in a JS, or to reference form data after a form is submitted.
If omitted, the value will not be sent at all 		<input type="text" id="fname" name="fname">
value Specify an initial value for an input field
(type: text, password, hidden)
Specify the text on the button
(type: button, reset, submit)
Specify the value associated with the input (also the value sent on submit)
(type: checkbox, radio, image) 		vehicle 1
readonly Specify that an input field is read-only
- The value will be sent when submitting the form!
- Useful to keep a user from changing the value until some other conditions have been met.
Then, a JS can remove the readonly value, and make the input field editable.
disabled Specify that an input field should be disabled
- The value will not be sent when submitting the form
- Useful to keep a user from using the <input> until some other condition has been met.
Then, a JS could remove the disabled value, and make the <input> usable-->
size Specify the visible width, in characters, of an input field (default: 20)
types: text, search, tel, url, email, password
height|width always Specify the height and width of an <input>
Types: image
min|max Specify the minimum|maximum values
types: number, range, date, datetime-local, month, time, week
minlength/maxlength Specify the minimum|maximum characters allowed
Type: text, search, url, tel, email, password
Not provide any feedback => use JS if needed
multiple Specify that more than one value is allowed
Type: email, file
Tip: For type="file": Use CTRL or SHIFT key while selecting
Tip: For type="email": Separate emails with a comma.
pattern Specify a regular expression that the input field's value is checked against, when the form is submitted
Types: text, date, search, url, tel, email, password see more
Tip: Use the global title attribute to describe the pattern to help the user
placeholder Specify an expected value of an input field (displayed in the field)
Types: text, search, url, tel, email, password
required Specify that an input field is required
Types: text, search, url, tel, email, password, date pickers, number, checkbox, radio, file
step Specify the legal number intervals for an input field
Types: number, range, date, datetime-local, month, time, week
autofocus Automatically get focus when the page loads



list Refer to a <datalist> that contains pre-defined options for an <input>
list value = datalist id
autocomplete Specify autocomplete is on or off (default: on)
<form> <input> types: text, search, url, tel, email, password, datepickers, range, and color
possible to have autocomplete "on" for the form, "off" for specific input fields, or vice versa

Type: image

Type: image

Form* attributes for <input>

Elements Explaination Example
form Specify the form the <input> belongs to
form value = id value

The field below is outside the form element, but still part of the form.
formaction Specify the URL of the file that will process the input when submitted
Override the action attribute of <form>
Type: submit, image


formenctype Specify how the form-data should be encoded when submitted (only for forms with method="post")
Override the enctype attribute of <form>
Type: submit, image

formmethod Specify the HTTP method for sending form-data to the action URL
Override the method attribute
Type: submit, image


formtarget Specify where to display the response
Override the target attribute
Type: submit, image
formtarget value = target value


formnovalidate Specify that an <input> should not be validated when submitted
Override the novalidate attribute
Type: submit
