URL Encode

Keywords Explaination Example
URL Uniform Resource Locators: another word for a web address
Composed of words (easier to remember)
Ex: w3schools.com
Composed of an Internet Protocol (IP) address
Ex: 192.68.20.50
Syntax scheme://prefix.domain:port/path/filename
scheme: Define the type of Internet service
most common: http or https
prefix: Define a domain prefix
default for http: www
domain: Define the Internet domain name
Ex: w3schools.com
port: Define the port number at the host
default for http: 80
path: Define a path at the server
If omitted: the root directory of the site
filename: Define the name of a document or resource
Common Schemes
  • http (HyperText Transfer Protocol): Common web pages. Not encrypted
  • https (Secure HyperText Transfer Protocol): Secure web pages. Encrypted
  • ftp (File Transfer Protocol): Downloading or uploading files
  • file: A file on your computer

HTML or XHTML

Elements Explaination Example
General EXtensible HyperText Markup Language
  • XHTML is a stricter, more XML-based version of HTML
  • XHTML is HTML defined as an XML application
  • XHTML is supported by all major browsers
Restriction
  • <!DOCTYPE> required
  • xmlns attribute in <html>required
  • <html>, <head>, <title>, <body> required
  • Elements must always be properly nested
  • Elements must always be closed
  • Elements must always be in lowercase
  • Attribute names must always be in lowercase
  • Attribute values must always be quoted
  • Attribute minimization is forbidden
