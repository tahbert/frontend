General JS variables can be converted to a new variable or another data type:
  1. Automatically by itself
  2. By the use of a function
String to Number
Number() Convert a value to a number if possible
Number("3.14");
Number(" ");
Number("") ;
Number("99 88");
parseInt() Parse a string and return the first integer recognised
parseInt("10")
parseInt("10.00")
parseInt("10.33")
parseInt("34 45 66")
parseInt(" 60 ")
parseInt("40 years")
parseInt("He was 40");

parseFloat() Parse a string and return the first floating point number recognised
parseFloat(10);
parseFloat("10");
parseFloat("10.33");
parseFloat("34 45 66");
parseFloat("He was 40");
Variable to Number
Unary + Operator Convert a variable to a number
let x = "5";
let y = +x;

let a = "Tahbert";
let b = +a;
Numbers to Strings
String() String(value)
Return a string
String(123);
String(100 + 23);
toString() number.toString(radix)
Return a string
(123).toString();
(100+23).toString();
(100+23).toString(2);
toExponential() number.toExponential(x) (x: optional, an integer between 0 and 20)
Return a string,
With a number rounded and written using exponential notation.
(5.56789).toExponential();
(5.56789).toExponential(3);
(5.56789).toExponential(2);
toFixed() number.toFixed(x) (x: optional, number of decimals, default: 0)
Return a string,
With a number rounded and written with a specified number of decimals.
(5.56789).toFixed();
(5.56789).toFixed(3);
(5.56789).toFixed(2);
toPrecision() number.toPrecision(x)
(x: optional, number of digits.If omitted, the number is returned without any formatting.)
Return a string,
With a number written with a specified length
(5.56789).toPrecision();
(5.56789).toPrecision(3);
(5.56789).toPrecision(2);

Automatic Type Conversion
JS automatically calls the variable's toString() function
when you try to "output" an object or a variable
{name:"Tahbert"};
[1,2,3,4];
new Date();
Numbers and booleans are also converted, but this is not very visible
123;
true;
false;
When JS operate on a "wrong" data type,
it will try to convert the value to a "right" type
 // The result is not always what you expect:
5 + null;
"5" + null;
"5" + 2;
"5" - 2;
"5" * "2";
