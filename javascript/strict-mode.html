Strict Mode

Strict Mode
Elements Explaination Visual Code
Useful Why "strict mode" is useful:
  • Throw an error
  • Make it easier to write "secure" JS
  • Change previously accepted "bad syntax" into real errors.
  • Make it impossible to accidentally create a global variable.
Declaring At the beginning of a script => Global scope "use strict";
myFunction();

function myFunction() {
 y = 3.14; // error
}
At the beginning of a function => Local scope x = 3.14; // not cause an error
myFunction();

function myFunction() {
"use strict";
 y = 3.14; // error
}
Not allowed
in strict mode
Using a variable, without declaring it "use strict";
x = 3.14; // error
Using an object, without declaring it "use strict";
x = {p1:10, p2:20}; // error
Deleting a variable (or object) "use strict";
let x = 3.14;
delete x; // error
Deleting a function "use strict";
function x(p1, p2) {};
delete x; // error
Duplicating a parameter name "use strict";
function x(p1, p1) {}; // error
Octal numeric literals "use strict";
let x = 010; // error
Octal escape characters "use strict";
let x = "\010"; // error
Writing to a read-only property "use strict";
const obj = {};
Object.defineProperty(obj, "x", {value:0, writable:false});
obj.x = 3.14; // error
Writing to a get-only property "use strict";
const obj = {get x() {return 0} };
obj.x = 3.14; // error
Deleting an undeletable property "use strict";
delete Object.prototype; // error
The word eval|arguments cannot be used as a variable: "use strict";
let eval = 3.14; // error
with statement "use strict";
with (Math){x = cos(2)}; // error
eval() is not allowed to create variables in the scope from which it was called: "use strict";
eval ("let x = 2");
alert (x); // error
this keyword: If the object is not specified,
- normal mode: functions will return the global object (window)
- strict mode: functions will return undefined
 "use strict";
function myFunction() {
 alert(this); // return undefined
}
myFunction();
