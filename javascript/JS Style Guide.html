Style Guide

Coding Conventions

Elements Explaination Example
Space space around operators ( = + - * / ) let x = y + z;
space after commas const arr = ["Volvo", "Saab", "Fiat"];
Indentation 2 spaces function myFn() {
 return "Hi";
}
Statement

simple statement

  • end with semicolon
 const person = {fname:"Tahbert", lname:"Nguyen"};

complex statement

  • open bracket: end the first line, 1 space before
  • close bracket: new line, without leading spaces
  • end without semicolon

function

function myFn() {
 return 'Hi Tahbert';
}

loop

for (let i = 0; i < 5; i++) {
 x += i;
}

condition:

if (time < 20) {
 greeting = "Good day";
} else {
 greeting = "Good evening";
}
Object

short object

  • property-value pair, no space between
 const person = {fname:"John", lname:"Doe"};

long object

  • opening bracket: on the same line as the object name
  • closing bracket: new line, without leading spaces
  • property-value pair: 1 space between
 const person = {
 fname: "John",
 lname: "Doe",
 age: 50,
 eyeColor: "blue"
};
Long Line advoid lines longer than 80 characters
break after an operator or a comma. 		document.getElementById("demo").innerHTML =
"Hello Dolly.";
Identifier
  • variable and func in camelCase
  • constant (like PI) in UPPERCASE
  • undercore is not allowed in JS => affect id/class name in HTML
 lastName = "Doe";
Loading JS in HTML simple syntax for loading external scripts
type attribute is not necessary 		<script src="myscript.js"></script>
File Names lower case if possible
some web servers are case sensitve, some aren't

Best Practices

Elements Explaination Example
Variable avoid global variables and functions
Declaration should be at the top of each script or function
Initialize Variable should be initialized when declaring
Declare Object with const to prevent any accidential change of type
Declare Arrays with const to prevent any accidential change of type
Automatic Type Conversions beaware of Automatic Type Conversions
A variable can change its data type
Comparison === operator forces comparison of values and type
Parameter Defaults argument missed => value undefined
=> good practice to assign default values to arguments
Switches with Defaults always end switch statements with a default
even if there is no need for it
Primative Values or Objects JS not treat numbers, strings, or booleans as objects
Slow down execution speed, and produces nasty side effects
eval() avoid eval()
