Events

Keywords Explaination Example
Interface: Event
change when the value of an element has been changed.
when the checked state has been changed (radio, checkbox)
    differ from oninput
  • oninput immediately when changing value
  • onchange when the element loses focus
  • onchange also works on <select>
 el.onchange = () => {
 const x = document.getElementById("mySelect").value;
 alert("You select: " + x)
}
Interface: InputEvent
input when <input> or <textarea> value is modified el.oninput = () => {
 const x = document.getElementById("myInput").value;
 console.log("You wrote: " + x)
}
beforeinput when <input> or <textarea> value is about to be modified
also applies to elements with contenteditable enabled
and to any element when designMode is turned on.
 later
Interface: DragEvent > Fired on the draggable target
dragstart when the user starts to drag an element or text selection
cancelable
el.addEventListener("dragstart", (event) => {
 event.target.classList.add("dragging");
});
drag when an element or text selection is being dragged
while dragging, fires every 350 milliseconds.
 el.addEventListener("drag", (event) => {
 console.log("dragging");
});
dragend when the user has finished dragging an element or text selection
cancelable
el.addEventListener("dragend", (event) => {
 event.target.classList.remove("dragging");
});
Interface: DragEvent > Fired on the drop target
dragenter when a dragged element or text selection enters a valid drop target.
about to enter a drop target.
 later
dragover when an element or text selection is being dragged over a valid drop target
fires every 350 milliseconds.
can't be dropped by default => use preventDefault()
 later
dragleave when a dragged element or text selection leaves a valid drop target
about to leave a drop target.
 later
drop when an element or text selection is dropped on a valid drop target later
