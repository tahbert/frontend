Spaces Spaces around operators ( = + - * / ) let x = y + z;
Spaces after commas const myArray = ["Volvo", "Saab", "Fiat"];
Indentation 2 spaces function toCelsius(fahrenheit) {
 return (5 / 9) * (fahrenheit - 32);
}
Statement Simple Statement:
  • Ending with semicolon
 const person = {firstName: "John", lastName: "Doe"};
Complex Statement
  • Opening bracket: end the first line, 1 space before
  • Closing bracket: new line, without leading spaces
  • Ending without semicolon
 // Function
function toCelsius(fahrenheit) {
 return (5 / 9) * (fahrenheit - 32);
}
// Loops
for (let i = 0; i < 5; i++) {
 x += i;
}
// Conditionals:
if (time < 20) {
greeting = "Good day";
} else {
greeting = "Good evening";
}
Object Short Object
  • Property-value pair, no space between
 const person = {firstName:"John", lastName:"Doe"};
Long Object
  • Opening bracket: on the same line as the object name
  • Closing bracket: new line, without leading spaces
  • Property-value pair: 1 space between
 const person = {
 firstName: "John",
 lastName: "Doe",
 age: 50,
 eyeColor: "blue"
};
Long Line Avoid lines longer than 80 characters, break after an operator or a comma. document.getElementById("demo").innerHTML =
"Hello Dolly.";
Naming
  • Variables and Functions in camelCase
  • Constants (like PI) in UPPERCASE
  • Undercore is not allowed in JS => affect id/class name in HTML
 lastName = "Doe";

const obj = getElementById("demo_1")
Loading JS in HTML Simple syntax for loading external scripts
(type attribute is not necessary) 		<script src="myscript.js"></script>
File Extensions .js extension
File Names Lower case file names (if possible)
(some web servers are case sensitve, some aren't)
Best Practices
Elements Explaination Visual Code
Variables Global variables:
Global variables and functions can be overwritten by other scripts => avoid
Local Variables: always declare with the var keyword or the let keyword
Declaration Should be at the top of each script or function.
Initialize Variables Should be initialized when you declaring
Declare Objects Declaring with const to prevent any accidential change of type
Declare Arrays Declaring with const to prevent any accidential change of type
new Object() Don't Use new Object()
Automatic Type Conversions Beware of Automatic Type Conversions
A variable can change its data type
Comparison Using === operator forces comparison of values and type
Parameter Defaults Missing argument, the value is set to undefined
=> It's a good practice to assign default values to arguments.
Switches with Defaults Always end switch statements with a default
Even if you think there is no need for it.
Primative Values or Objects JS not treat numbers, strings, or booleans as objects
Slow down execution speed, and produces nasty side effects
eval() Unnecessary to use eval(). It allows arbitrary code to be run, it also represents a security problem.
