Hoisting JS's default behavior of moving declarations to the top => a variable can be used before it has been declared.
warning always declare variables at the beginning of every scope to avoid bugs
not allowed in strict mode
var keyword: can initialize a value x = 5; // Initialize 5 to x
var x; // Declare x
let keyword: can't initialize a value => not work x = 5; // Initialize 5 to x
let x; // ReferenceError
const keyword: can't initialize a value => not work x = 5; // Initialize 5 to x
const x; // Not run
